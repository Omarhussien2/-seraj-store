This page documents the auxiliary backend services that provide configuration, media management, AI-driven interaction, and administrative analytics for the Seraj Store. These APIs bridge the gap between the static frontend and the dynamic operational requirements of the store.

## Configuration & Admin Settings

The configuration system provides the frontend with essential environmental data (like WhatsApp contact info) and dynamic business logic (like shipping fees). It uses a fallback mechanism that prefers values stored in the `SiteContent` collection over environment variables.

### Data Flow: Config Retrieval
The `GET /api/config` route is called by the SPA on initialization to populate global state.

1.  **Environment Fallbacks**: Loads defaults from `NEXT_PUBLIC_SHIPPING_FEE` and `NEXT_PUBLIC_FREE_SHIPPING_ABOVE` [src/app/api/config/route.ts:8-9]().
2.  **Database Override**: Attempts to fetch `shipping_fee` and `free_shipping_above` from the `SiteContent` model [src/app/api/config/route.ts:13-20]().
3.  **Static Env Injection**: Merges these with static environment variables for payment and contact [src/app/api/config/route.ts:28-34]().

### Admin Settings Management
The `GET/PUT /api/admin/settings` route allows administrators to modify these values at runtime without redeploying.

*   **GET**: Fetches current shipping settings from the DB [src/app/api/admin/settings/route.ts:10-29]().
*   **PUT**: Updates or creates (`upsert: true`) keys in the `SiteContent` collection [src/app/api/admin/settings/route.ts:58-63]().

**Sources:** [src/app/api/config/route.ts:1-37](), [src/app/api/admin/settings/route.ts:1-74](), [src/app/admin/page.tsx:64-88]()

## Media Upload Pipeline

Seraj Store utilizes Cloudinary for all media hosting. There are two distinct upload paths: one for administrators (products/articles) and one for public users (Story Wizard).

### Admin Multi-File Upload (`/api/upload`)
Requires `requireAdmin` middleware [src/app/api/upload/route.ts:37-38](). It supports both images and videos with parallel processing.

*   **Validation**: Limits to 20 files per request [src/app/api/upload/route.ts:65](). Enforces 10MB for images and 50MB for videos [src/app/api/upload/route.ts:22-23]().
*   **Transformations**: 
    *   **Images**: Resized to 1200x1200px max with `auto:good` quality [src/app/api/upload/route.ts:114-117]().
    *   **Videos**: Encoded to H.264 MP4 asynchronously [src/app/api/upload/route.ts:122-126]().

### Public Child Photo Upload (`/api/upload-child-photo`)
Used by the Story Wizard. This endpoint is public but strictly rate-limited and validated to prevent abuse.

*   **Rate Limiting**: 20 uploads per 10 minutes per IP using `isRateLimited` [src/app/api/upload-child-photo/route.ts:20-26]().
*   **Strict Types**: Only `JPEG`, `PNG`, and `WebP` allowed [src/app/api/upload-child-photo/route.ts:40-49]().
*   **Storage**: Files are stored in the `seraj/children-photos` folder with an 800x800px limit [src/app/api/upload-child-photo/route.ts:66-72]().

**Sources:** [src/app/api/upload/route.ts:1-161](), [src/app/api/upload-child-photo/route.ts:1-94](), [src/lib/rateLimit.ts:1-30]()

## AI Chat: Mama Zainab

The "Ask Zainab" (اسأل زينب) feature provides a streaming AI interface with a specific Egyptian grandmother persona.

### Implementation Details
The route uses a multi-provider fallback strategy to ensure high availability.

1.  **Persona**: Defined by a comprehensive `SYSTEM_PROMPT` that enforces Egyptian slang, 3-5 line brevity, and Islamic parenting values [src/app/api/chat/route.ts:32-70]().
2.  **Rate Limiting**: In-memory `rateMap` restricts users to 30 messages per hour per IP [src/app/api/chat/route.ts:7-21]().
3.  **Context**: Includes the last 10 messages from `history` for context-aware responses [src/app/api/chat/route.ts:111-116]().
4.  **Fallback Chain**:
    *   **DeepSeek**: Primary provider (OpenAI-compatible) [src/app/api/chat/route.ts:128-141]().
    *   **Gemini (Key 1 & 2)**: Secondary/Tertiary fallbacks [src/app/api/chat/route.ts:144-150]().

## Administrative Statistics

The `GET /api/stats` endpoint provides a high-level overview of store performance for the Admin Dashboard. It utilizes a single MongoDB `$facet` aggregation to compute multiple metrics in one pass.

### Aggregation Pipeline
The pipeline calculates the following [src/app/api/stats/route.ts:17-44]():
*   **totalOrders**: Count of all documents in the `Order` collection.
*   **pendingOrders**: Count where `orderStatus` is "pending".
*   **pendingStories**: Count where `customStory.heroName` exists and the order is not yet completed.
*   **totalRevenue**: Sum of the `total` field across all orders.
*   **recentOrders**: The 5 most recent orders with projected fields for the dashboard table.

## Testimonials API

The testimonials system manages customer feedback displayed on the landing page.

*   **Model**: `ITestimonial` includes fields for `avatarInitials` and `avatarColor` to generate consistent UI avatars without requiring user photos [src/lib/models/Testimonial.ts:3-15]().
*   **GET /api/testimonials**: Returns active testimonials by default, sorted by a manual `order` field [src/app/api/testimonials/route.ts:14-15]().
*   **POST /api/testimonials**: Admin-only route to create new entries [src/app/api/testimonials/route.ts:27-45]().

**Sources:** [src/lib/models/Testimonial.ts:1-36](), [src/app/api/testimonials/route.ts:1-46]()
