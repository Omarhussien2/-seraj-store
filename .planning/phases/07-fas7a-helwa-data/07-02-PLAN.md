1. Create `src/lib/models/Place.ts` following EXACT pattern from Product.ts:

   ```typescript
   // Interface IPlace (like IProduct)
   export interface IPlace extends Document {
     name_ar: string;
     name_en: string;
     description_short: string;
     area: string;
     city: string;             // "Cairo", "Alexandria", "Sharm El Sheikh", etc.
     address: string;
     location: {
       lat: number;
       lon: number;
     };
     min_price: number;
     max_price: number;
     price_range_id: number;   // 1-5 scale
     min_age: number;
     max_age: number;
     avg_duration_hours: number;
     is_free: boolean;
     indoor_outdoor: "indoor" | "outdoor" | "mixed" | "unknown";
     booking_required: boolean;
     website_url: string;
     external_source: string;  // "Kidzapp" | "Manual" | "Admin"
     external_detail_url: string;
     phone: string;
     facebook_url: string;
     instagram_url: string;
     category_ids: number[];   // [1, 3] = Fun&Play + Outdoor
     image_url: string;
     last_price_update: Date;
     active: boolean;          // soft delete like Product
     order: number;            // sort order like Product
     createdAt: Date;
     updatedAt: Date;
   }
   ```

   Schema details:
   - `name_en`: required, indexed with text (for search)
   - `name_ar`: required, indexed with text
   - `city`: required, indexed (for filtering)
   - `location`: GeoJSON Point schema with `lat`/`lon` — also add `2dsphere` index for future geo-queries
   - `category_ids`: [Number], indexed
   - `is_free`: Boolean, indexed
   - `indoor_outdoor`: enum, indexed
   - `active`: Boolean, default true, indexed (like Product)
   - `external_source`: string
   - `last_price_update`: Date
   - Text index: `{ name_en: "text", name_ar: "text", city: "text", area: "text" }`
   - Compound index: `{ active: 1, city: 1, category_ids: 1 }`
   - Export pattern: `mongoose.models.Place || mongoose.model<IPlace>("Place", PlaceSchema)` — prevents Next.js hot-reload duplicate model error
   - `{ timestamps: true }` like all existing models

2. Create `src/app/api/places/route.ts` following EXACT pattern from products/route.ts:

   Add at top: `export const runtime = "nodejs"; export const dynamic = "force-dynamic";`

   **GET /api/places** (PUBLIC — no auth):
   - Query params: ?city=Cairo&category=3&is_free=true&indoor_outdoor=outdoor&q=search&all=true&limit=20&page=1
   - Build filter object: { active: true } by default (or all if ?all=true for admin)
   - city: exact match
   - category: category_ids contains the number (use $in)
   - is_free: boolean match
   - indoor_outdoor: exact match
   - q: text search using $text with $search
   - Pagination: page + limit, return { data, count, page, totalPages }
   - Sort: { order: 1 } by default
   - Return: `{ success: true, count, data }` (same envelope as products)

   **POST /api/places** (ADMIN only):
   - Call `requireAdmin()` first — return auth error if not admin
   - Zod schema validating all Place fields (follow CreateProductSchema pattern)
   - Check no duplicate name_en
   - `Place.create(validated)`
   - Return 201 with created place

3. Create `src/app/api/places/[id]/route.ts`:

   Add at top: `export const runtime = "nodejs"; export const dynamic = "force-dynamic";`

   **GET /api/places/[id]** (PUBLIC):
   - Find by `_id` param, only if active:true (unless ?all=true)
   - Return 404 if not found

   **PATCH /api/places/[id]** (ADMIN only):
   - requireAdmin()
   - Zod schema for partial update (all fields optional)
   - FindByIdAndUpdate with `{ new: true, runValidators: true }`
   - Return updated place

   **DELETE /api/places/[id]** (ADMIN only):
   - requireAdmin()
   - Soft delete: set `active: false` (same pattern as Product)
   - Return 200 with message

4. Run `npx next build` to verify no TypeScript errors
