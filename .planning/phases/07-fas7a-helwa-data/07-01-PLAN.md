<task type="auto">
  <name>Task 1: Build Kidzapp API Scraper + Shared Types</name>
  <files>scripts/kidzapp-types.ts, scripts/scrape-kidzapp-egypt.ts, data/fas7a-helwa-kidzapp.csv</files>
  <action>
1. Create `scripts/kidzapp-types.ts` with TypeScript interfaces:
   - `KidzappExperience` — full API response shape (id, uid, name, title, title_en, description, address, location:{lat,lon}, phone, website, instagram_link, image_url, image_carousel_list, venue_type[], price_range, booking_required, ages_display[], working_hours[], categories[], primary_category, area:{id,name_en}, city:{id,name_en,name_ar}, price[]{id,type_en,type_ar,final_price,currency}, price_age_groups, price_age_prices, created_at, url, booking_url)
   - `KidzappPaginatedResponse` — { count, next: string|null, previous: string|null, results: KidzappExperience[] }
   - `PlaceRow` — the unified place schema matching CSV + future MongoDB model:
     * id: number
     * name_ar: string
     * name_en: string
     * description_short: string
     * area: string
     * city: string
     * address: string
     * lat: number | null
     * lon: number | null
     * min_price: number
     * max_price: number
     * price_range_id: number (1-5)
     * min_age: number
     * max_age: number
     * avg_duration_hours: number
     * is_free: boolean
     * indoor_outdoor: "indoor"|"outdoor"|"mixed"|"unknown"
     * booking_required: boolean
     * website_url: string
     * external_source: string
     * external_detail_url: string
     * phone: string
     * facebook_url: string
     * instagram_url: string
     * category_ids: string (semicolon-separated)
     * image_url: string
     * last_price_update: string (YYYY-MM-DD)
   - `VENUE_TYPE_MAP` const: ["indoors"]→"indoor", ["outdoors"]→"outdoor", both→"mixed", []→"unknown"
   - `CATEGORY_MAP` const: Map Kidzapp IDs → local IDs {66517:1, 66514:2, 66518:3, 66512:4, 37:5, 66513:1, 38:3, 49:3, 45:1, 48:3, 66516:6}
   - `LOCAL_CATEGORIES` const: {1:"Fun & Play/لعب و مرح", 2:"Shows & Cinema", 3:"Outdoor & Parks", 4:"Art & Educational", 5:"Farms & Animals", 6:"Eat Out"}
   - Export all types so Plan 02's model can import from here

2. Create `scripts/scrape-kidzapp-egypt.ts` that:
   - Uses native `fetch()` to call `https://api.kidzapp.com/api/3.0/experiences/?country_code=eg&page_size=100`
   - Follows `data.next` pagination until all 615 fetched (7 pages, 200ms delay between pages)
   - Filters to `type === "venue"` AND `show === true` only
   - Transforms each to PlaceRow:
     * name_ar: exp.name (English OK)
     * name_en: exp.name
     * description_short: first 120 chars of exp.description OR exp.top_tip, clean \r\n→spaces
     * area: exp.area?.name_en || ''
     * city: exp.city?.name_en || ''
     * address: exp.address || ''
     * lat/lon: exp.location?.lat || null, exp.location?.lon || null
     * min_price/max_price: from exp.price[].final_price > 0, or parse price_age_prices text
     * is_free: all prices 0 or empty
     * indoor_outdoor: VENUE_TYPE_MAP[exp.venue_type]
     * website_url: exp.website || ''
     * external_source: "Kidzapp"
     * external_detail_url: "https://kidzapp.com/" + exp.url
     * phone: exp.phone || ''
     * facebook_url: exp.website if contains "facebook" else ''
     * instagram_url: exp.instagram_link ? "https://instagram.com/" + exp.instagram_link : ""
     * category_ids: exp.categories.map(c => CATEGORY_MAP[c.id]).filter(Boolean).join(";")
     * image_url: exp.image_url || ''
     * last_price_update: exp.created_at.split('T')[0] (preserves 2019 dates — TRUST)
   - Uses `json2csv` to write CSV with BOM prefix to `data/fas7a-helwa-kidzapp.csv`
   - Prints summary with city/category breakdown
   - Install: `npm install --save-dev json2csv`

3. Run: `npx tsx scripts/scrape-kidzapp-egypt.ts`
  </action>
  <verify>Console shows "Fetched 615 experiences", 400+ venues written.
   `data/fas7a-helwa-kidzapp.csv` has correct header, last_price_update shows 2019 dates, external_source="Kidzapp", lat/lon columns populated.</verify>
  <done>CSV with 400+ Kidzapp Egypt venues matching PlaceRow schema with GPS, real price dates, and source attribution</done>
</task>
