  <files>scripts/kidzapp-types.ts, scripts/scrape-kidzapp-egypt.ts, data/fas7a-helwa-kidzapp.csv</files>
  <action>
1. Create `scripts/kidzapp-types.ts` with TypeScript interfaces:
   - `KidzappExperience` — maps full API response (id, name, title, title_en, description, address, location, phone, website, instagram_link, image_url, image_carousel_list, venue_type, price_range, booking_required, ages_display, working_hours, categories, primary_category, area, city, price[], price_age_groups, price_age_prices, created_at)
   - `KidzappPaginatedResponse` — { count, next, previous, results[] }
   - `Fas7aHelwaRow` — matches exact CSV schema columns: id, name_ar, name_en, description_short, area, address, min_price, max_price, price_range_id, min_age, max_age, avg_duration_hours, is_free, indoor_outdoor, booking_required, website_url, external_source, external_detail_url, phone, facebook_url, instagram_url, category_ids, last_price_update
   - `VENUE_TYPE_MAP` const: ["indoors"] → "indoor", ["outdoors"] → "outdoor", both → "mixed", empty → "unknown"
   - `CATEGORY_MAP` const: Map Kidzapp category IDs to simplified local IDs (1=Fun & Play, 2=Shows/Cinema, 3=Outdoor/Parks, 4=Educational/Museums, 5=Farms/Animals)

2. Create `scripts/scrape-kidzapp-egypt.ts` that:
   - Uses native `fetch()` (Node 22 built-in) to call `https://api.kidzapp.com/api/3.0/experiences/?country_code=eg&page_size=100`
   - Follows `data.next` pagination links until all 615 experiences are fetched (7 pages)
   - Adds 200ms delay between pages to be respectful
   - Filters to only `type === "venue"` AND `show === true` (skip events/courses)
   - For each experience, transforms to Fas7aHelwaRow:
     * `name_ar`: Use `exp.name` (English is fine per user decision — translate later)
     * `name_en`: `exp.name`
     * `description_short`: Take first 120 chars of `exp.description` OR `exp.top_tip` if description is empty. Clean up \r\n to spaces.
     * `area`: `exp.area?.name_en || exp.city?.name_en || ''`
     * `address`: `exp.address || ''`
     * `min_price` / `max_price`: Extract from `exp.price[]` array — filter `final_price > 0`, take min/max. If `price[]` is empty, parse `exp.price_age_prices` text field (split by \r\n, extract EGP numbers).
     * `price_range_id`: `exp.price_range || 1`
     * `min_age`: `Math.min(...exp.ages_display)` or 0 if empty
     * `max_age`: `Math.max(...exp.ages_display)` or 100 if empty
     * `avg_duration_hours`: Estimate 3 if "full day" in price types, 2 if "2 hours", 3 default for outdoor, 2 for indoor
     * `is_free`: true if ALL prices are 0 or price array is completely empty and price_age_prices has no positive numbers
     * `indoor_outdoor`: Map from `exp.venue_type[]` using VENUE_TYPE_MAP
     * `booking_required`: `exp.booking_required`
     * `website_url`: `exp.website || ''`
     * `external_source`: `"Kidzapp"`
     * `external_detail_url`: `"https://kidzapp.com/" + exp.url`
     * `phone`: `exp.phone || ''`
     * `facebook_url`: `exp.website` if it contains "facebook.com", else `''`
     * `instagram_url`: `exp.instagram_link ? "https://instagram.com/" + exp.instagram_link : ""`
     * `category_ids`: Map `exp.categories[].id` through CATEGORY_MAP, join with semicolons
     * `last_price_update`: `exp.created_at.split('T')[0]` — This preserves the ORIGINAL date from Kidzapp (mostly 2019) so users can see when prices were last confirmed. This is a TRUST feature — showing "2019-04-04" is better than showing today's date for old data.
   - Uses `json2csv` to convert rows to CSV with BOM prefix (`\uFEFF`) for Arabic compatibility
   - Writes to `data/fas7a-helwa-kidzapp.csv`
   - Prints summary: total fetched, total venues, cities breakdown, categories breakdown
   - Install json2csv as devDependency: `npm install --save-dev json2csv && npm install --save-dev @types/json2csv`

3. Run the script: `npx tsx scripts/scrape-kidzapp-egypt.ts`
