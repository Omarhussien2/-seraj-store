- صفحة "عالم ماما" موجودة في `public/index.html` (سطر 900) مع 3 تابات: مقالات | فسحة حلوة | اسألي زينب
- تاب المقالات فيه **4 كروت placeholder ثابتة** مغطاة بـ "coming soon" overlay

## القرار: MongoDB وليس JSON Files

### لماذا MongoDB وليس ملفات JSON؟

| المعيار | ملف JSON واحد | ملف JSON لكل مقال | MongoDB |
|---------|---------------|-------------------|---------|
| إضافة مقال من الأدمن | مستحيل بدون deploy | مستحيل بدون deploy | من لوحة التحكم مباشرة |
| بحث بالعربي | تحميل كل الملفات + فلترة JS | تحميل كل الملفات + فلترة JS | MongoDB text index |
| فلترة (قسم + عمر) | تحميل كل شي ثم فلترة | تحميل كل شي ثم فلترة | query-level filtering |
| Pagination | مستحيل فعلياً | مستحيل فعلياً | `.skip().limit()` |
| تعديل مقال | deploy جديد | deploy جديد | لحظي من الأدمن |
| الأداء مع 100+ مقال | بطيء (ملف ضخم) | 100+ HTTP request | استعلام واحد سريع |
| رفع صور المقالات | لا يمكن ربطها | لا يمكن ربطها | Cloudinary URL في الـ DB |

**القرار: MongoDB** — نفس الـ DB المستخدمة للمنتجات والطلبات. الملف JSON الحالي يستخدم كـ **seed script فقط** لتعبئة البيانات الأولية.

## Phase 1: Database Model

### [NEW] `src/lib/models/Article.ts`

```ts
const ArticleSchema = new mongoose.Schema({
  // ---- Identity ----
  slug:           { type: String, required: true, unique: true, index: true },
  title:          { type: String, required: true },
  seoTitle:       { type: String },  // عنوان مختلف لـ SEO لو محتاج

  // ---- Classification ----
  section:        { type: String, required: true },
  // e.g. "الحمل والرضاعة", "من الولادة إلى سنتين", "الشاشات والإنترنت"...
  // هذا هو التصنيف الرئيسي من الفهرس (الـ 12 قسم)
  ageGroup:       { type: String },
  // e.g. "الحمل", "0-2", "2-5", "5-10", "متنوع"
  tags:           [{ type: String }],
  // e.g. ["عند", "سلوك", "٣-٦ سنين"] — للبحث والفلترة الدقيقة

  // ---- Content ----
  excerpt:        { type: String, required: true },
  // ملخص قصير (سطرين) يظهر في كارت المقال
  contentMarkdown: { type: String, required: true },
  // المقال كامل بصيغة Markdown — يتحول لـ HTML في الفرونت

  // ---- Media ----
  coverImage:     { type: String },
  // URL صورة الغلاف (Cloudinary) — تظهر في الكارت وأعلى المقال
  coverImageAlt:  { type: String, default: "" },

  // ---- Sources / References ----
  sources: [{
    label:  { type: String, required: true },
    // e.g. "CDC — تربية إيجابية"
    url:    { type: String },
    // e.g. "https://www.cdc.gov/..." — اختياري، لو مصدر أوفلاين يكون فاضي
    note:   { type: String },
    // e.g. "تم الرجوع إليه في ابريل ٢٠٢٦" — ملاحظة اختيارية
  }],

  // ---- Meta ----
  readingTime:    { type: Number, default: 5 },
  // بالدقائق — يتحسب تلقائي من طول المحتوى أو يدوي
  author:         { type: String, default: "فريق سراج" },
  publishedAt:    { type: Date },
  // تاريخ النشر — المقالات بدون تاريخ = مسودة
  active:         { type: Boolean, default: true, index: true },
  order:          { type: Number, default: 0 },
  // ترتيب العرض داخل القسم

  // ---- SEO ----
  metaDescription: { type: String },
  // وصف مخصص للـ meta tag — لو فاضي يستخدم excerpt
}, { timestamps: true });

// Full-text search index for Arabic content
ArticleSchema.index({ title: "text", excerpt: "text", contentMarkdown: "text", tags: "text" });

// Compound index for listing queries
ArticleSchema.index({ active: 1, section: 1, order: 1 });
```

### لماذا هذا التصميم؟

1. **`section` بدل `category`**: الفهرس الحالي فيه 12 قسم واضح — هذا هو التصنيف الأساسي
2. **`tags` مصفوفة حرة**: تسمح بفلترة متعددة الأبعاد (عمر + موضوع + كلمات مفتاحية)
3. **`sources[]` مصفوفة**: المقال الأول يحتوي مراجع `[web:373]` — نحتاج نظام مصادر حقيقي
4. **`contentMarkdown`**: أسهل للكتابة والتعديل من HTML، والتحويل لـ HTML يتم في الفرونت
5. **`publishedAt` كـ Date**: يسمح بالجدولة — مقال بتاريخ مستقبلي = مسودة تلقائية
6. **`coverImage` URL وليس ملف**: نفس نظام Cloudinary المستخدم في المنتجات

## Phase 2: Backend API

### [NEW] `src/app/api/articles/route.ts`

#### `GET /api/articles` — عام (Public)

```
Query params:
  ?section=الحمل والرضاعة    — فلترة بالقسم
  ?ageGroup=2-5               — فلترة بالمرحلة العمرية
  ?tag=عند                    — فلترة بتاج معين
  ?search=النوم               — بحث نصي (MongoDB text search)
  ?page=1&limit=12            — pagination (default: page=1, limit=12)
  ?all=true                   — عرض كل المقالات بما فيها غير النشطة (admin)

Response (listing — بدون contentMarkdown لتوفير bandwidth):
{
  "success": true,
  "data": [...],          // مصفوفة المقالات (بدون contentMarkdown)
  "pagination": {
    "page": 1,
    "limit": 12,
    "total": 40,
    "pages": 4
  },
  "sections": [           // قائمة الأقسام المتاحة مع العدد
    { "name": "الحمل والرضاعة", "count": 4 },
    { "name": "من الولادة إلى سنتين", "count": 4 },
    ...
  ]
}
```

**ملاحظة مهمة**: الـ GET لا يرجع `contentMarkdown` في الـ listing — فقط `title`, `excerpt`, `section`, `coverImage`, `readingTime`, `slug`. هذا يقلل حجم الـ response من ~200KB لـ ~15KB.

#### `POST /api/articles` — أدمن فقط
- `requireAdmin()` + Zod validation
- يحسب `readingTime` تلقائياً من طول `contentMarkdown` (متوسط ~200 كلمة عربية/دقيقة)
- يولّد `slug` تلقائي من العنوان لو لم يُعطى

### [NEW] `src/app/api/articles/[slug]/route.ts`

#### `GET /api/articles/:slug` — عام
- يرجع المقال كامل **بما فيه** `contentMarkdown` و `sources`
- يستخدمه الفرونت لعرض صفحة المقال الواحد

#### `PATCH /api/articles/:slug` — أدمن فقط
- تعديل أي حقل في المقال
- Zod validation

#### `DELETE /api/articles/:slug` — أدمن فقط
- Soft delete (`active: false`)

### Cache Headers
```
GET /api/articles        → Cache-Control: public, s-maxage=120, stale-while-revalidate=300
GET /api/articles/:slug  → Cache-Control: public, s-maxage=300, stale-while-revalidate=600
```

### نظام الفلترة والبحث — UX

#### 1. أقسام الفهرس (Section Chips)
- الـ 12 قسم من الفهرس تظهر كـ chips أفقية scrollable
- الضغط على قسم = فلتر بالـ `section`
- "الكل" يعرض كل المقالات
- كل chip يعرض عدد المقالات: `الحمل والرضاعة (4)`
- **مصدر البيانات**: الـ `sections` array المرجع من الـ API

#### 2. البحث النصي
- Input مفعّل مع debounce (300ms)
- يبحث في العنوان والملخص والمحتوى والتاجز
- يرسل `?search=` للـ API (MongoDB text search بالعربي)
- أثناء البحث: يخفي الـ chips ويعرض "نتائج البحث عن: ___"
- لو مفيش نتائج: رسالة "مفيش مقالات بالبحث ده — جربي كلمات تانية"

#### 3. Pagination
- **"عرض المزيد"** button بدل numbered pagination — أبسط UX للمحتوى العربي
- أول تحميل: 12 مقال
- كل ضغطة "عرض المزيد": 12 مقال إضافي
- يختفي الزر لما يخلّصوا

### Markdown → HTML Conversion
- نستخدم مكتبة خفيفة مثل **marked** (8KB gzipped) في الفرونت
- تتحول `# عنوان` → `<h1>`, `## عنوان` → `<h2>`, إلخ
- القوائم والروابط والنصوص المنسقة تشتغل تلقائي
- يُضاف كـ `<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>`
- بديل: تحويل Markdown لـ HTML في الـ API (backend) — أثقل لكن أأمن

### عرض المصادر
- section مخصص في آخر المقال بعنوان "المصادر والمراجع"
- كل مصدر يظهر كـ row:
  - **اسم المصدر** (عريض)
  - **رابط** (لو موجود) — أيقونة ↗ + رابط قابل للضغط يفتح في تاب جديد
  - **ملاحظة** (لو موجودة) — نص صغير رمادي
- لو مفيش مصادر، الـ section لا يظهر

### مقالات ذات صلة
- تُجلب من الـ API: مقالات من نفس الـ `section` (بحد أقصى 3)
- تظهر كـ كروت صغيرة أفقية في آخر المقال

## Phase 6: Seed Script

### [NEW] `scripts/seed-articles.ts`
- يقرأ ملف الفهرسة JSON الحالي
- يحوّل كل مقال لـ document بالشكل الصحيح:
  - `id` → يُتجاهل (MongoDB يولّد `_id`)
  - `title` → `title`
  - `seo_title` → `seoTitle`
  - `slug` → `slug`
  - `category` → يُستخدم لتحديد الـ `section` من الفهرس
  - `age_group` → `ageGroup`
  - `excerpt` → `excerpt`
  - `content_markdown` → `contentMarkdown`
  - يستخرج المراجع `[web:XXX]` من المحتوى ويحولها لـ `sources[]`
  - يحسب `readingTime` تلقائي
- **Upsert by slug**: لو المقال موجود لا يُعاد كتابته (يحافظ على تعديلات الأدمن)
- يُشغَّل مرة واحدة: `npx tsx scripts/seed-articles.ts`
