# خارطة تطوير باك اند الأدمن

هدف هذه القائمة هو تحويل الأدمن من لوحة تشغيل بسيطة إلى مركز تحكم كامل في محتوى وتجربة المتجر، خصوصًا checkout والرسائل والإعدادات المتغيرة.

## أولويات قريبة

## تم تنفيذه في هذه الجولة

- إعدادات أدمن إضافية عبر `/api/admin/settings` للتحكم في نص زر "كمل تسوق"، جملة موعد وصول الطلب، وتفعيل/إخفاء بوت سراج حسب الصفحات.
- إخفاء بوت سراج تلقائيًا في صفحات حساسة مثل checkout/success/wizard/preview مع إمكانية تعديل القائمة من الداشبورد.
- دعم رفع أكثر من صورة للطفل أثناء رحلة القصة المخصصة، وتخزين `photoUrls` داخل الطلب مع الحفاظ على `photoUrl` للتوافق القديم.
- عرض روابط تحميل مباشرة لصور الطفل في تفاصيل الطلب داخل الأدمن.

1. **إدارة Checkout من الأدمن**
   - إعداد حقول صفحة checkout: العناوين، النصوص المساعدة، رسائل الخطأ، نص زر التأكيد.
   - إعداد طرق الدفع: InstaPay، QR، روابط الدفع، ورسائل التعليمات.
   - إعداد حقول اختيارية/إجبارية بدون تعديل الكود.

2. **إدارة التسعير والشحن**
   - توسيع إعدادات الشحن الحالية إلى مناطق/محافظات.
   - دعم قواعد شحن مجانية حسب المنتجات أو قيمة الطلب.
   - سجل تغييرات للإعدادات حتى يعرف الأدمن من غيّر ماذا ومتى.

3. **تحسين إدارة الكوبونات**
   - دعم أكثر من قاعدة خصم من واجهة الأدمن، وليس قاعدة واحدة فقط.
   - شاشة تفاصيل كوبون تعرض الطلبات التي استخدمته.
   - فلترة حسب نشط/منتهي/مستنفد، وتكرار كوبون موجود بسرعة.

4. **Content Blocks قابلة للتحكم**
   - تحويل نصوص checkout والسلة والنجاح إلى مفاتيح داخل `SiteContent`.
   - إضافة preview داخل الأدمن قبل النشر.
   - دعم حفظ مسودة ثم نشر.

5. **تقارير الأدمن**
   - تقارير كوبونات: إجمالي الخصومات، أفضل الأكواد، معدل التحويل.
   - تقارير الطلبات: الإيرادات قبل/بعد الخصم، الشحن، المنتجات الأكثر طلبًا.
   - تصدير CSV للفترات الزمنية.

## تحسينات تقنية مهمة

- نقل منطق تسعير الطلب إلى خدمة واحدة مشتركة بين `/api/orders` و`/api/coupons/validate` لتقليل التكرار.
- إضافة اختبارات unit لمنطق الخصم وحدود الاستخدام.
- إضافة اختبارات integration لمسار إنشاء الطلب مع كوبون صالح/منتهي/مستنفد.
- حماية API الأدمن بصلاحيات أكثر تفصيلًا عندما يظهر أكثر من دور إداري.
- دراسة نقل الواجهة العامة من `public/app.js` إلى React تدريجيًا، أو على الأقل فصل checkout إلى module مستقل لتسهيل الصيانة.

## ديون تقنية لو ظهرت مرة أخرى

- `middleware` deprecated في Next 16 ويجب التخطيط للهجرة إلى `proxy`.
- lint الكامل يحتوي أخطاء قديمة في ملفات scripts وadmin أخرى؛ لا تعتبر فشل lint الكامل دليلًا على فشل تغييرات الكوبونات وحدها.
- بعض نصوص الأدمن القديمة ظهرت سابقًا بترميز غير واضح في PowerShell، لذلك يفضل تعديل النصوص العربية عبر editor/patch مع مراجعة نهائية في المتصفح.
